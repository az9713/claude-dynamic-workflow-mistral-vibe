# Type-Escape Sweep Report — Mistral Vibe `vibe/core/`

> Generated by a 19-agent dynamic workflow (9 finders → 9 adversarial verifiers → 1 synthesis). 353 escapes verified. Skeptic default was REMOVABLE; an escape had to be *argued* into NECESSARY with a failed re-typing sketch. Verdicts reflect the codebase at generation time — confirm `file:line` before acting.

## 0. What this is, and how it was produced

**What a type-escape sweep is.** Vibe is checked by Pyright in *strict* mode — a type checker that verifies type
consistency before the code runs. A **type escape** is any place the code deliberately tells the checker to stop
checking: `cast(...)`, `# type: ignore` / `# pyright: ignore`, an `Any` annotation, or a 3-arg
`getattr(x, "name", default)`. Each is a blind spot — sometimes unavoidable, often a latent bug. A **sweep** is
the *exhaustive* audit of all of them: find every escape, then judge whether each is genuinely necessary or could
be re-typed properly. Coverage, not sampling.

**Why this was run as a workflow.** The sweep is hundreds of small, independent judgment tasks (one per escape)
plus a single consolidation — too much volume for one agent to read without its context degrading into
summary, and the judgment must be *adversarial*, which only works if the skeptic judging an escape did not also
find it. A Claude Code dynamic workflow matches that shape directly: nine **finder** agents catalogue escapes in
parallel (each reads only its slice of `vibe/core/`, so none loses focus); nine **adversarial verifier** agents —
separate contexts, so the check is genuinely independent — re-open the files and rule each escape REMOVABLE or
NECESSARY, defaulting to REMOVABLE and forced to sketch a concrete re-typing before allowing "necessary"; one
**synthesis** agent then sees every verdict at once and produces the ranked, clustered fix-list below. A bare
`grep` would find the lines but make no judgment; a single chat turn would make judgments but drown in volume and
could not be adversarial with itself. (The full design rationale is in
[`type-escape-sweep-workflow-writeup.md`](./type-escape-sweep-workflow-writeup.md).)

## 1. Summary

The sweep covers **353 adversarially-verified type escapes** across `vibe/core/`. Of these, **272 are REMOVABLE (77%)** and **81 are NECESSARY (23%)**. By kind: **`Any` — 278 total** (235 removable / 43 necessary); **`getattr` — 39** (23 / 16); **`cast` — 29** (13 / 16); **`type-ignore` — 5** (1 / 4); **`other` — 2** (0 / 2). The removable population is dominated by JSON/serialization boundaries that only flow into `.get()`, `isinstance`-narrowing, or pydantic `model_validate`/`model_dump` — all of which accept `object`/`dict[str, object]`/`JsonValue`. The largest single concentration is `nuage/remote_workflow_event_translator.py` (48 removable), followed by `llm/backend/anthropic.py` (34). The necessary residue clusters around three irreducible roots: third-party SDK contracts (MCP before-validators, Mistral SDK enums, foreign LLM message objects), the `BaseTool.get_parameters -> dict[str, Any]` base contract, and TypeVar-recovery casts in `tools/base.py`/`tools/ui.py` that no static mechanism can bridge.
